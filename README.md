# Skills Generator

**Agent Skills を作成・改善・診断・ローカライズするメタスキル**

AIエージェントが「新しいスキルを作って」という指示を受けたとき、単に推測で書き始めるのではなく、このリポジトリ内のベストプラクティスやテンプレートをすべて読み込み、プロの「スキルアーキテクト」として動作します。どれだけ複雑な要件であっても、実用的で安定したスキルパッケージを生み出します。

---

## ディレクトリ構成

```
skills-generator/
├── SKILL.md              # メイン指示書：4つのモードを駆動する「脳」
├── assets/               # 出力用テンプレート集
│   ├── skill-template.md         # スキルのひな型
│   ├── audit-report-template.md  # 監査レポートテンプレート
│   └── improvement-report-template.md  # 改善レポートテンプレート
├── references/           # 専門知識とルールのデータベース
│   ├── best-practices.md         # 確実に発火させるためのルール集
│   ├── security-checklist.md     # セキュリティ診断チェックリスト
│   ├── script-patterns.md        # ヘルパースクリプトの実装パターン
│   ├── skill-standard-summary.md # スキル標準仕様のまとめ
│   ├── implementation-patterns-guide.md  # 実装パターンガイド
│   └── improve-checklist.md      # 改善チェックリスト
└── examples/             # 理想的なスキルの完成例
    └── example-skill/
        ├── SKILL.md
        ├── assets/
        │   └── review-template.md
        └── references/
            └── review-checklist.md
```

プロンプトを単一ファイルに詰め込むのではなく、役割ごとにフォルダを分割して**外部参照（リファレンス）として細かく設計・パッケージング**するのが、Agent Skills における実装の王道パターンです。

## 導入手順

### Step 1: 環境別セットアップ（保存先の確認）

Agent Skills は、使用する AI エディタ・ツールによって読み込まれるディレクトリが異なります。プロジェクトのルートディレクトリに以下のフォルダを作成してください。

#### GitHub Copilot

- **利用ディレクトリ**: `.github/skills/`
- **設定**: `.vscode/settings.json` に以下を追加します。

```json
{
  "chat.useAgentSkills": true
}
```

#### GPT Codex（VS Code 拡張・CLI）

- **利用ディレクトリ**: `.agents/skills/`
- **設定**: 不要。ディレクトリが存在するだけで自動認識されます。

#### Google Gemini（AntiGravity）

- **利用ディレクトリ**: `.agent/skills/`（複数形の「s」なし）
- **設定**: 不要。ディレクトリが存在するだけで自動認識されます。

> [!NOTE]
> GPT Codex は `.agents`（複数形）、Google Gemini は `.agent`（単数形）と異なる点に注意してください。

---

### Step 2: リポジトリのクローン

作成した `skills` ディレクトリ内で、本リポジトリをクローンして配置します（下記の例は GitHub Copilot 環境の場合）。

```bash
cd .github/skills
git clone https://github.com/safubuki/skills-generator.git
```

ZIP でダウンロードして解凍し、`skills-generator` フォルダとして配置しても構いません。

これだけで導入完了です。

## 動作モード

Skills Generator は、ユーザーの目的に応じて **4つのモード** を使い分けます。

| モード | 説明 | 使用例 |
|--------|------|--------|
| **Generate（新規作成）** | テンプレートとベストプラクティスを参照し、構造化されたスキルパッケージを一から構築。ヘルパースクリプト付きスキルの生成にも対応 | 「pytestのスキルを作って」「自動化スクリプト付きで」 |
| **Improve（改善）** | 既存スキルの発火キーワードやディレクトリ構成を評価・最適化して改善案を提示 | 「スキルを改善して」「発火条件を良くして」「うまく発火しない」 |
| **Audit（監査・診断）** | 外部から取得した未知スキルのコードを読み込み、データの外部送信・破壊的操作などのセキュリティリスクを厳しくチェック | 「スキルをチェックして」「スキルを監査」 |
| **Localize（ローカライズ）** | 英語スキルのコード構造やコマンドを壊すことなく、日本語での指示・発火キーワードに安全に翻訳 | 「スキルを日本語化して」「翻訳して」 |

また、「こんなことしたい」「〜を自動化したい」と漠然とした要望を伝えた場合でも、目的を分析して最適なスキル構成を提案・生成します。

## クイックスタート

導入後、AI チャットで以下のように話しかけるだけでスキルが起動します。

### Generate — 具体的なプロンプト例

漠然と「スキルを作って」と伝えるよりも、**やりたいこと・読み込みたいファイル・出力に含めたい内容** を一緒に伝えると、より実用的なスキルが生成されます。

**シンプルな例（まず試してみたい方向け）:**

```
フロントエンドプロジェクト向けの「README生成スキル」を作って。
package.json の依存関係を読み込み、セットアップ手順・依存ライブラリ一覧を
自動生成する実用的なドキュメントスキルにしてほしい。
```

**要件を詳しく伝えた例（より実践的なスキルを作りたい方向け）:**

```
フロントエンドプロジェクト向けの「README生成スキル」を作って。
単なる概要だけでなく、以下を満たす実践的なドキュメントスキルにしてほしい。

1. package.json の依存関係を抽出してライブラリ一覧を記載する
2. docs/architecture/adr/ 内の最新ADRを読み込み、「設計思想」セクションに要約する
3. .github/workflows/deploy.yml のデプロイ先環境変数（STG/PRD）を抽出して
   アクセス可能なURLをリストアップする
4. TODO コメントが残っているコードがあれば、警告文を太字でREADMEの先頭に挿入する
```

> [!TIP]
> 最初はシンプルな要件から始め、生成されたスキルを確認してから **Improve モード** で段階的に育てていくのがおすすめです。

## セキュリティについて

インターネット上には悪意ある指示が含まれた「野良スキル」が存在する場合があります。
外部スキルを利用する際は、必ず **Audit モード** でセキュリティ診断を実施することを強く推奨します。
本リポジトリはその診断プロセスも透明化するために、ソースをすべて公開しています。
